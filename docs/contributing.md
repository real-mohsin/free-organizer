# Contributing to Free Organizer

Thank you for your interest in contributing!

We welcome bug fixes, improvements, new features, documentation updates, and UI enhancements.

---

# Getting Started

1. Fork the repository.
2. Clone your fork.
3. Install dependencies.

npm install

4. Start the development server.

npm run dev

---

# Branch Naming

feature/add-client-page

feature/dashboard-chart

fix/login-validation

---

# Commit Message Convention

feat: Add client search

fix: Resolve payment table bug

docs: Update installation guide

style: Improve button spacing

refactor: Simplify dashboard logic

---

# Coding Standards

- TypeScript only
- Avoid using `any`
- Keep components reusable
- Keep functions small and focused
- Write meaningful variable names
- Prefer composition over duplication

---

# Folder Naming

Folders

kebab-case

Components

PascalCase

Variables

camelCase

Constants

UPPER_CASE

---

# Pull Requests

Before opening a Pull Request:

- Ensure the project builds successfully.
- Follow the existing code style.
- Test your changes.
- Update documentation if needed.

---

# Reporting Bugs

Please include:

- Description
- Steps to reproduce
- Expected behavior
- Screenshots (if applicable)

---

# Feature Requests

Explain:

- Problem
- Proposed Solution
- Alternative Solutions

---

# Code of Conduct

Be respectful.

Be constructive.

Help others learn.

Everyone is welcome.
